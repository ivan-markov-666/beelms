# MVP функционалности на QA4Free по екрани и модули

_Роля: PM / Architect. Мост между PRD и реалния backlog за MVP на ниво екрани и поведения._

Този документ описва минимално необходимите функционалности за **MVP** версията на QA4Free,
групирани по екрани и модули. Той стъпва на:
- `idea/01.defina-idea`
- `idea/02.developmentProcess.md`
- `docs/architecture/system-architecture.md`

Целта е да служи като мост между продуктова идея, архитектура и реален backlog.

## 1. Публична Wiki (без акаунт)

### 1.1. Начален екран / Списък със статии
- Преглед на списък с основните Wiki статии.
- Търсене по заглавие / ключова дума.
- Филтриране по език (напр. BG / EN / DE).
- Навигация към конкретна статия.

### 1.2. Екран „Wiki статия“
- Преглед на съдържанието на статията.
- Превключване на език (ако статията съществува на повече от един език).
- Показване на базова информация (заглавие, последна редакция).

## 2. Акаунт и профил

### 2.1. Регистрация
- Форма за регистрация с имейл и парола.
- Валидация на парола (минимални правила за сигурност).
- Съхранение на потребителя в системата.

### 2.2. Вход / Изход
- Форма за вход с имейл и парола.
- Създаване на сесия / JWT токен при успешен вход.
- Изход (инвалидация на сесията/токена).

### 2.3. Забравена парола
- Форма за заявка за възстановяване на парола (имейл адрес).
- Изпращане на имейл с линк/токен за смяна на паролата.
- Екран за въвеждане на нова парола.

### 2.4. Профил и управление на акаунта
- Екран „Моят профил“ с основна информация.
- Функция за **деактивиране/изтриване на акаунта** в съответствие с GDPR:
  - Потвърждение преди изтриване.
  - Анонимизиране на личните данни.
- Функция за **експорт на лични данни** (файл за сваляне).

## 3. Практическа среда – UI демо

### 3.1. Екран с UI елементи за упражнения
- Страница с различни UI компоненти:
  - бутони;
  - drop-down списъци;
  - текстови полета;
  - чекбоксове и радио бутони;
  - други базови HTML елементи.
- Ясно описание на целта на екрана – за manual и UI automation тестване.
- Възможност за **reset** на състоянието на задачата (връщане в начално състояние).
- Няколко примерни задачи (описание какво да се постигне чрез взаимодействие с елементите).

## 4. Практическа среда – API демо

### 4.1. Swagger UI за QA4Free Demo API
- Достъпен **Swagger/OpenAPI** интерфейс за примерен REST API.
- Описание на основните endpoints (CRUD операции върху примерни ресурси).
- Възможност потребителят да изпраща заявки директно през Swagger UI.

- **За текущото MVP** Demo API е реализиран като минимален Training API (ping/echo). Пълният CRUD демо ресурс остава за бъдеща версия (post-MVP).

### 4.2. Упражнения за API / integration тестване
- Описани примерни сценарии:
  - създаване, четене, обновяване и изтриване на ресурс (CRUD);
  - позитивни и негативни тестове (невалидни данни, неоторизиран достъп и др.).
- Възможност резултатите (успех/грешка, статус код) да се използват за обучение.

## 5. Администраторски панел

### 5.1. Управление на Wiki съдържание
- Екран за списък с Wiki страници.
- Създаване и редакция на статии (заглавие, съдържание, език).
- Деактивиране/скриване на статии, без да се трият окончателно.

### 5.2. Управление на версии на Wiki
- Преглед на историята на версиите за дадена страница.
- Преглед на разлики (diff) между две версии.
- Връщане към предишна версия (rollback).
- Изтриване на избрани стари версии (освен текущата активна).

### 5.3. Управление на потребители
- Списък с потребители (основна информация).
- Активиране/деактивиране на акаунти.

### 5.4. Базово табло с метрики (MVP ниво)
- Брой регистрирани потребители.
- Най-преглеждани Wiki страници.
- Брой изпълнени практичeски задачи (агрегирано).

## 6. Невидими, но задължителни функционалности (cross-cutting)

### 6.1. Мултиезичност
- Поддръжка на поне два езика в интерфейса (напр. BG и EN) за MVP.
- Преводаемо съдържание на ключови UI елементи и Wiki статии.

### 6.2. GDPR и управление на данни
- Права на потребителя:
  - достъп до лични данни;
  - изтриване (right to be forgotten);
  - преносимост на данните;
  - коригиране на неточни данни.
- Техническа имплементация:
  - изтриване/анонимизиране на данни при закриване на акаунт;
  - експорт на данни;
  - политика за съхранение на данни.

### 6.3. Метрики и анализи
- Събиране на базови метрики за:
  - потребителско поведение (страници, време, навигация);
  - учебен напредък (завършени практичeски задачи);
  - производителност на системата (време за отговор, грешки).

### 6.4. Безопасност
- JWT автентикация и защита на сесиите.
- Основни защити:
  - CSRF, XSS, SQL injection;
  - защита срещу брут форс (rate limiting, временни блокировки).
