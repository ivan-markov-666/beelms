# Training API – примерни заявки\n\nTraining API-то е лек NestJS service, използван за упражнения по API/automation.\n\n- Base URL (dev): `http://localhost:4000/api/training`\n- Няма authentication – endpoint-ите са публични в dev среда.\n\n> За формалните дефиниции виж `docs/architecture/openapi.yaml` (Training секцията).\n\n---\n\n## 1. Ping – GET /training/ping\n\nНай-простият health/ping endpoint. Ползва се за базови проверки и примери в документацията.\n\n```bash\ncurl -X GET "http://localhost:4000/api/training/ping"\n```\n\nОчакван отговор (примерен):\n\n```json\n{\n  "message": "pong",\n  "timestamp": "2025-01-10T10:00:00.000Z"\n}\n```\n\n---\n\n## 2. Echo – POST /training/echo\n\nВръща обратно стойността, която е изпратена в заявката, заедно с метаданни (`receivedAt`, `requestId`). Подходящо за упражнения по:\n
- сериализация/десериализация на JSON;\n- валидиране на contract между клиент и сървър;\n- basic API тестове.\n\n### 2.1. Пример със string\n\n```bash\ncurl -X POST \\\n  "http://localhost:4000/api/training/echo" \\\n  -H "Content-Type: application/json" \\\n  -d '{\n    "value": "Hello QA4Free"\n  }'\n```\n\nПримерен отговор:\n\n```json\n{\n  "value": "Hello QA4Free",\n  "receivedAt": "2025-01-10T10:05:00.000Z",\n  "requestId": "f7c4..."\n}\n```\n\n### 2.2. Пример с object\n\n```bash\ncurl -X POST \\\n  "http://localhost:4000/api/training/echo" \\\n  -H "Content-Type: application/json" \\\n  -d '{\n    "value": {\n      "id": 123,\n      "name": "Sample",\n      "tags": ["api", "training"]\n    }\n  }'\n```\n\nПримерен отговор:\n\n```json\n{\n  "value": {\n    "id": 123,\n    "name": "Sample",\n    "tags": ["api", "training"]\n  },\n  "receivedAt": "2025-01-10T10:06:00.000Z",\n  "requestId": "5a21..."\n}\n```\n\n### 2.3. Възможни грешки\n\n- `400 Bad Request` – когато полето `value` липсва или типът му е невалиден според OpenAPI (`string | number | boolean | object | array`).\n\n---\n\nТези примери са синхронизирани с `docs/architecture/openapi.yaml` и NestJS имплементацията в `training-api/src`. При промяна на Training API първо актуализирайте OpenAPI, след това примерите тук.\n
