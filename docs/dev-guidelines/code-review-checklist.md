# Code Review Checklist

## Общи
- [ ] Описанието на PR-а пояснява целта и контекста
- [ ] Всички тестове минават и билдът е зелен

## Коректност и качество
- [ ] Логиката е правилна, обхванати са edge cases
- [ ] Няма оставени `TODO`/`FIXME` в production код

## Стил и консистентност
- [ ] Lint проверките (ESLint) минават без предупреждения
- [ ] Приложен е форматът на Prettier

## Типова безопасност
- [ ] Няма "any" типове без обосновка
- [ ] Спазени са strict null checks и опционални properties са проверени

## Тестване
- [ ] Добавени/обновени са unit/integration тестовете
- [ ] Покритието е ≥ 80 % за променените файлове

## Сигурност и поверителност
- [ ] Не са качени secrets или чувствителни данни
- [ ] Входовете се валидират, изходите се енкодират
- [ ] Личните данни се обработват в съответствие с GDPR

## Производителност
- [ ] Няма O(N²) алгоритми в критични пътища
- [ ] SQL заявките са индексирани и оптимизирани

## Международизация (i18n)
- [ ] Всички нови UI низове имат ключове и преводи
- [ ] Проверени са липсващи ключове с `npm run i18n:check`
- [ ] API/DB промени поддържат `lang` / `Accept-Language`

## Документация
- [ ] Публичните API промени са документирани
- [ ] Коментиран е сложният код (JSDoc/TypeDoc)

## Deployment
- [ ] CI pipeline минава успешно
- [ ] Миграциите са обратно съвместими

## Одобрение
- [ ] Минимум един рецензент е одобрил
- [ ] Commit съобщенията следват Conventional Commits
