## Summary
Като **потребител/курсист** искам **Practice UI страница с различни UI елементи (формуляри, бутони, таблици и др.)**, за да мога да практикувам manual тестване и UI automation върху реалистична, но безопасна среда.

## Acceptance Criteria
- Съществува публична страница (напр. `/practice/ui-demo`), достъпна без специални роли.
- Страницата съдържа набор от UI елементи, например:
  - текстови полета, текстови области;
  - бутони (вкл. disabled/primary/secondary състояния);
  - checkbox-и, radio бутони;
  - селект/дропдаун;
  - таблица/лист с елементи.
- Страницата предоставя **минимални инструкции/описание** за това какви упражнения могат да се правят (напр. в отделен текстов блок "Задача: ...").
- По желание и/или за интеграция с Tasks API:
  - UI-то може да зарежда текстово описание на задача чрез `GET /api/tasks/{id}` (примерна UI задача) и да го визуализира;
  - не е задължително в WS-7 да има пълен интегриран flow за submit/оценка директно през UI – важното е да има реални елементи и сценарии за упражнение.
- Страницата е стилизирана консистентно с останалия FE (layout, typography) и поддържа минимум BG/EN за ключовите етикети, в синхрон с i18n слоя.
- Има поне един FE тест, който проверява:
  - че основните UI елементи са визуализирани;
  - че текстовото описание/инструкции за задачата се рендерират правилно.
